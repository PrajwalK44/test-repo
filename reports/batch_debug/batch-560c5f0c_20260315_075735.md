# Incident Report: Batch `560c5f0c` (Authentication Failure in `bcrypt.checkpw`)

---

### **1. Issue**
The application encountered a **`TypeError` during user authentication** when `bcrypt.checkpw()` was called. The error occurred because:
- `bcrypt.checkpw()` expects **both arguments (`password` and `hash`) to be `bytes`**, but the stored `user.password_hash` (retrieved from the database) was a **`str`** (not encoded).
- The plaintext password (`data["password"]`) was correctly encoded to `bytes` via `.encode("utf-8")`, but the comparison failed due to the type mismatch with `user.password_hash`.
- **Symptoms**:
  - Login requests with valid credentials would silently fail (return `401 Unauthorized`).
  - No explicit error in logs (handled gracefully but incorrectly).
  - Detected during batch `5196dd60` debugging (incident `DEV-74`).

---

### **2. Solution**
The fix ensured **type consistency** by encoding the stored `password_hash` (from `str` to `bytes`) before passing it to `bcrypt.checkpw()`.
- **Root Cause**: The ORM (or database layer) returned `password_hash` as a `str`, but `bcrypt` operations require `bytes`.
- **Fix**: Added `.encode("utf-8")` to `user.password_hash` in the `login()` route.

---

### **3. Applied Approach**
1. **Reproduced the Issue**:
   - Simulated a login request with valid credentials.
   - Confirmed `bcrypt.checkpw()` returned `False` due to `TypeError` (implicitly caught).
2. **Debugged Type Mismatch**:
   - Inspected `type(user.password_hash)` → Confirmed it was `str`.
   - Verified `bcrypt.checkpw()` signature requires `bytes` for both arguments.
3. **Applied Fix**:
   - Modified `auth.py` to encode `user.password_hash` to `bytes` before comparison.
4. **Validated**:
   - Re-ran login tests with the same credentials → Success (`200 OK`).

---

### **4. Files Modified**
- `python-service/app/routes/auth.py` (Line 55)

---

### **5. Exact Code Modified**
#### **Before (Broken)**
```python
is_valid = bcrypt.checkpw(
    data["password"].encode("utf-8"),  # bytes (correct)
    user.password_hash                  # str (incorrect)
)
```

#### **After (Fixed)**
```python
is_valid = bcrypt.checkpw(
    data["password"].encode("utf-8"),  # bytes
    user.password_hash.encode("utf-8") # str → bytes (fixed)
)
```
