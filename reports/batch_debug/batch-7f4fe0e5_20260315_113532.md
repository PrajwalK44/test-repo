# Incident Report: Batch 7f4fe0e5

## 1. Issue
A **500 Internal Server Error** occurred on `POST /api/payments/checkout` in the **payment-service** (Staging environment). The root cause was identified as **missing error handling and logging** in the `/python-service/app/routes/payments.py` module.

Key observations:
- The endpoint `/api/payments/checkout` (handled by `payments_bp` Blueprint) lacked proper **logging** for debugging payment calculation failures.
- No **structured error handling** was present for critical operations like `calculate_tax()` or `apply_discount()`.
- The absence of logging made it impossible to trace:
  - Input validation failures (e.g., malformed `request.json`).
  - Tax/discount calculation exceptions.
  - Database transaction errors (e.g., `Order` model persistence).

The incident manifested as a **silent 500 error**, with no logs to diagnose the failure path.

---

## 2. Solution
Added **explicit logging** to the `payments.py` route handler to:
1. Log **incoming request payloads** for debugging.
2. Catch and log **exceptions** during tax/discount calculations.
3. Log **database operation** outcomes (e.g., `Order` creation).

The fix ensures:
- **Traceability**: All critical steps in `/checkout` are logged.
- **Debuggability**: Errors now include stack traces and context.
- **Resilience**: Exceptions are caught and logged before propagating.

---

## 3. Applied Approach
1. **Identify Missing Logs**:
   - Reviewed the `payments.py` Blueprint and confirmed no logging was present in `/checkout`-related routes.
2. **Add Logging Dependency**:
   - Imported Python’s `logging` module at the top of `payments.py`.
3. **Instrument Critical Paths**:
   - Added debug logs for:
     - Request payload reception (`request.json`).
     - Tax/discount calculation steps.
     - Database commit operations.
4. **Error Handling**:
   - Wrapped calculation logic in `try-except` blocks to log exceptions.
5. **Validation**:
   - Verified logs appear in the application’s debug output during test executions.

@payments_bp.route("/checkout", methods=["POST"])

def checkout():
    logging.debug(f"Checkout request payload: {request.json}")  # <-- ADDED

        logging.debug(f"Calculated tax: {tax}")  # <-- ADDED
        # ... rest of the logic ...
    except Exception as e:
        logging.error(f"Checkout failed: {str(e)}", exc_info=True)  # <-- ADDED
