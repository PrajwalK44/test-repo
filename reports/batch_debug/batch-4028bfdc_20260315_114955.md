# Incident Report: batch-4028bfdc

## 1. Issue
A **500 Internal Server Error** occurred on `POST /api/payments/checkout` in the staging environment. The root cause was traced to an **authentication subsystem failure** where `bcrypt.checkpw()` was being called with mismatched argument types:

- **Problem**: In `auth.py`, the `user.password_hash` (a string) was passed directly to `bcrypt.checkpw()` without UTF-8 encoding, while the input password was encoded. This caused a `TypeError` during byte comparison:
  ```python
  bcrypt.checkpw(
      data["password"].encode("utf-8"),  # bytes
      user.password_hash                   # str (not encoded)
  )
  ```
- **Impact**: All login attempts failed with a server error, blocking payment checkout flows that required authentication.

---

## 2. Solution
The fix ensured type consistency by encoding the stored `password_hash` (string) to UTF-8 bytes before passing it to `bcrypt.checkpw()`. This aligns with the input password’s encoding and resolves the `TypeError`.

---

## 3. Applied Approach
1. **Reproduced the Issue**:
   - Simulated a login request to `/api/auth/login` with valid credentials.
   - Confirmed the `500` error and traced logs to `bcrypt.checkpw()` failure.

2. **Root Cause Analysis**:
   - Inspected `auth.py` and identified the type mismatch in `bcrypt.checkpw()` arguments.
   - Verified that `user.password_hash` was a string (not bytes) via debug logs.

3. **Fix Implementation**:
   - Modified the `login()` function to encode `user.password_hash` to UTF-8 before comparison.

4. **Validation**:
   - Re-tested login flow with the same credentials.
   - Confirmed successful authentication and `200 OK` response.

## 5. Exact Code Modified
```diff
--- a/python-service/app/routes/auth.py
+++ b/python-service/app/routes/auth.py
@@ -52,7 +52,7 @@ def login():
     is_valid = bcrypt.checkpw(
         data["password"].encode("utf-8"),
-        user.password_hash
+        user.password_hash.encode("utf-8")
     )
```

**Before**: Type mismatch (`bytes` vs `str`) caused `bcrypt.checkpw()` to fail.
**After**: Both arguments are UTF-8 encoded `bytes`, enabling correct comparison.
