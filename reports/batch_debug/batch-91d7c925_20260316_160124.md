# Incident Report: Batch 91d7c925 (DEV-103)

## 1. Issue
The `test_payment_flow` CI test was failing due to incorrect tax calculation logic in `payment_service.py`. The original implementation used integer division (`//`) on the subtotal, which:
1. Truncated decimal values (e.g., `$123.45` → `123`)
2. Applied tax rate to the truncated value, causing under-calculation
3. Produced incorrect tax amounts for orders with cents (e.g., `$123.45` subtotal with 10% tax would calculate as `$12.00` instead of `$12.35`)

**Error Manifestation**:
- CI test `test_payment_flow` failed when validating tax calculations for orders with non-integer subtotals
- The bug affected all payment transactions with cents in the subtotal

## 2. Solution
Replaced the integer division logic with floating-point arithmetic to:
1. Preserve decimal precision in the subtotal
2. Apply the tax rate as a percentage (`TAX_RATE / 100`)
3. Round the result to 2 decimal places for currency compliance

The fix ensures tax is calculated on the *actual* subtotal (e.g., `$123.45 * 0.10 = $12.345` → `$12.35` after rounding).

## 3. Applied Approach
1. **Reproduced the Issue**:
   - Verified the failing test case with subtotal=`123.45` and `TAX_RATE=10`
   - Confirmed original code returned `$12.00` (incorrect) instead of `$12.35`

2. **Root Cause Analysis**:
   - Identified integer division (`//`) as the source of precision loss
   - Noted `TAX_RATE` was defined as an integer (e.g., `10` for 10%) but used incorrectly

3. **Fix Implementation**:
   - Replaced `int(subtotal) // 100 * TAX_RATE` with `subtotal * (TAX_RATE / 100)`
   - Added explicit rounding to 2 decimal places for currency formatting

4. **Validation**:
   - Re-ran `test_payment_flow` with edge cases:
     - Subtotals with cents (`123.45`, `99.99`)
     - Subtotals as integers (`100.00`)
     - Zero subtotal (`0.00`)

def calculate_tax(subtotal):
     """
     Calculate tax for a given subtotal.
     Args:
         subtotal: The order subtotal (float).
     Returns:
         The tax amount rounded to 2 decimal places.
     """
-    tax = int(subtotal) // 100 * TAX_RATE
+    tax = subtotal * (TAX_RATE / 100)
     return round(tax, 2)
