# Incident Report: Batch `5196dd60`

## 1. Issue
A **`TypeError`** occurred during user authentication in the `/login` endpoint due to **incompatible byte/string comparison** in `bcrypt.checkpw()`. The error manifested when:
- The `password` input (from request) was encoded to `bytes` via `.encode("utf-8")`.
- The stored `user.password_hash` (from database) was **not encoded**, causing `bcrypt.checkpw()` to fail with:
  ```
  TypeError: 'str' does not support the buffer interface
  ```
  **Root Cause**:
  The `password_hash` field retrieved from the database was a raw string, but `bcrypt.checkpw()` expects both arguments (`password` and `hash`) to be `bytes`-like objects.

---

## 2. Solution
The fix ensured **consistent byte encoding** for both arguments passed to `bcrypt.checkpw()` by:
1. Encoding the stored `user.password_hash` to `bytes` before comparison.
2. Maintaining the existing encoding of the input `password` (already `.encode("utf-8")`).

This aligns with `bcrypt`’s requirement for byte inputs and resolves the `TypeError`.

---

## 3. Applied Approach
1. **Reproduced the Issue**:
   - Simulated a login request with valid credentials.
   - Confirmed the `TypeError` in logs when `bcrypt.checkpw()` was called.

2. **Root Cause Analysis**:
   - Inspected the `user` model and confirmed `password_hash` was stored as a `str` (not `bytes`).
   - Verified `bcrypt.checkpw()` documentation requires `bytes` for both arguments.

3. **Fix Implementation**:
   - Modified `auth.py` to encode `user.password_hash` to `bytes` before passing to `bcrypt.checkpw()`.

4. **Validation**:
   - Re-tested login with the same credentials → success (no `TypeError`).
   - Verified no regression in other auth endpoints (e.g., `/register`).

---

## 5. Exact Code Modified
### Before (Problematic Code)
```python
is_valid = bcrypt.checkpw(
    data["password"].encode("utf-8"),
    user.password_hash  # <-- TypeError: 'str' passed instead of 'bytes'
)
```

### After (Fixed Code)
```python
is_valid = bcrypt.checkpw(
    data["password"].encode("utf-8"),
    user.password_hash.encode("utf-8")  # <-- Now both args are 'bytes'
)
```
